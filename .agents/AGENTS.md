# Специфические правила разработки CloudMusicPlayer

Этот документ дополняет глобальные правила системы и содержит контекст, уникальный для проекта CloudMusicPlayer.

## 1. Специфика YouTube и стриминга (Стандарты 2026 года)
* **Клиент YouTubeKit:** Всегда использовать прямую загрузку потоков через `YouTubeKit` без промежуточных серверов. Инициализировать клиентские сессии с регулярным обновлением visitor data.
* **Invidious (Поиск):** Выполнять параллельный опрос серверов (через `withTaskGroup` / race) с таймаутом не более 6 секунд. Побеждает первый успешный ответ.
* **Обработка обложек:** Качество обложек YouTube запрашивать по цепочке fallback: `maxresdefault` (проверяя ширину > 120px) -> `hqdefault` -> `mqdefault` -> `sddefault`. Кешировать изображения через `NSCache`.

## 2. Специфика UI/UX плеера (Премиальный виниловый дизайн)
* **Тонарм (TonearmView):** Реалистичный тонарм с базой, изогнутым рычагом (`Path` + `QuadCurve`) и картриджем. Управляется анимацией поворота при смене состояния воспроизведения (пауза: −5°, игра: +25°).
* **Спектрограф (RealtimeVisualizerView):** 28 вертикальных полос с физикой гравитации пиков (каждый столбец отображает текущее значение и медленно падающий пик белого цвета).
* **Тактильный отклик:** Использовать `HapticManager.shared.triggerImpact(style:)` при клике на кнопки плеера и переключение режимов.
