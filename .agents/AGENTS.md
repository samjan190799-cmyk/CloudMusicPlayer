# Правила разработки CloudMusicPlayer (Актуальность: 2026 год)

Этот документ содержит критические правила, стандарты кодирования и контекст для всех AI-агентов, работающих с проектом CloudMusicPlayer.

## 1. Технологический стек и окружение (Стандарты 2026 года)
* **Минимальная версия iOS:** iOS 16.0+
* **Язык программирования:** Swift 6.0 (используется строгий режим проверки конкурентности — Strict Concurrency)
* **UI-фреймворк:** SwiftUI с современным синтаксисом (макросы, typed throws, if-let shorthand)
* **Менеджер зависимостей:** Swift Package Manager (SPM)

## 2. Правила Swift 6 Concurrency и Асинхронности
* **Изоляция потоков:** Все UI-представления (View) и Observable-объекты, работающие с интерфейсом, должны быть изолированы на главном потоке с помощью `@MainActor`.
* **Async/Await:** Избегать старых callback-функций (closures) для асинхронного кода. Использовать нативный async/await везде, где это возможно.
* **Работа с TaskGroup:** При динамическом параллельном выполнении задач (например, опросе серверов) обязательно указывать явный тип возвращаемого значения в замыкании `group.addTask { ... }` во избежание неоднозначности вывода типов компилятором:
  ```swift
  group.addTask { [weak self] () -> [ResultType]? in
      // Логика
  }
  ```
* **Безопасность типов:** Модели данных, передаваемые между асинхронными контекстами, должны соответствовать протоколу `Sendable`.

## 3. Особенности работы с YouTube и медиа-потоками (Специфика 2026 года)
* **YouTube API & Сигнатуры:** В 2026 году YouTube применяет новые механизмы защиты медиа-потоков (PO Token, Visitor Data). При работе с `YouTubeKit` всегда инициализировать клиентские сессии с обновлением visitor data и использовать актуальные асинхронные методы расшифровки потоков.
* **Invidious API (Поиск):** Публичные инстансы Invidious часто бывают перегружены. Поиск должен выполняться **параллельно** (через `withTaskGroup` / гонку запросов) по списку живых серверов с коротким тайм-аутом (не более 6 секунд).
* **Кадрирование обложек:** Качество обложек YouTube запрашивать по цепочке fallback: `maxresdefault` (проверяя ширину > 120px) -> `hqdefault` -> `mqdefault` -> `sddefault`. Кешировать изображения через `NSCache` для оптимизации памяти.

## 4. UI/UX стандарты (Премиальный дизайн)
* Использовать тактильную отдачу (`HapticManager`) на все ключевые действия: выбор элементов, завершение загрузки, ошибки, открытие плеера.
* Использовать кастомный стиль кнопок `ScaleButtonStyle` для плавного микровзаимодействия при нажатии.
* Детализированный дизайн плеера (реалистичный тонарм, виниловые пластинки, спектрографы с физикой падения пиков) должен быть анимирован через плавные пружинные анимации (`.interactiveSpring` / `.spring`).
